## 一、总体概述

### 1.1 第一批裁剪范围

根据《插件渐进式裁剪总览方案》第 4.2 节，第一批裁剪对象为 **无反向依赖的独立插件**，风险等级为 ⚪ 零风险。

- **插件数量**：22 个
- **配套库**：1 个（modelinglib）
- **预计减少文件**：1,096 个

- **操作方式**：从 `plugins.pro` 和 `libs.pro` 的 SUBDIRS 中移除，然后物理删除源码目录

### 2.1 studiowelcome — Qt Design Studio 欢迎页

#### 功能说明

Qt Design Studio 定制的欢迎页面插件，与标准 `welcome` 插件使用相同的 Welcome 模式位（`Core::Constants::MODE_WELCOME`），但提供 Qt Design Studio 风格的 QML 界面。包括近期项目列表、教程入口、示例浏览等 QML 组件。

**裁剪原因**：本工程使用标准 `welcome` 插件，不需要 Qt Design Studio 品牌欢迎页。默认就是禁用状态（`DisabledByDefault: true`）。

#### 依赖关系

```
QTC_PLUGIN_DEPENDS: coreplugin, projectexplorer, qtsupport
QTC_LIB_DEPENDS:    extensionsystem, utils
反向依赖:           无
```

#### 代码结构

```
src/plugins/studiowelcome/          （93 个文件，406 行 C++）
├── studiowelcome_dependencies.pri   # 依赖声明
├── studiowelcome.pro                # 构建配置
├── StudioWelcome.json.in            # 插件元数据
├── studiowelcomeplugin.h            # 插件主类头文件
├── studiowelcomeplugin.cpp          # 插件主类实现
├── studiowelcome_global.h           # 导出宏定义
├── qlitehtml.qrc                    # Qt 资源文件
├── qml/                             # QML 界面文件
│   ├── main.qml                     # 主界面
│   ├── CustomTab.qml                # 自定义标签页
│   ├── ProjectsGrid.qml             # 项目网格
│   ├── RecentProjects.qml           # 近期项目
│   ├── ExampleItem.qml              # 示例项
│   ├── ...                          # 更多 QML 组件
│   └── welcomepage/                 # 子模块
└── images/                          # 图片资源
    ├── *.png                        # 按钮/图标
    └── ...
```

#### 删减方法

**状态**：构建入口已在记录 001 中移除，本次只需物理删除源码目录。

```bash
# 物理删除源码目录
rm -rf src/plugins/studiowelcome/
```

**无需修改任何 .pro 文件**（已在前序步骤完成）。

### 2.2 helloworld — 示例插件模板

#### 功能说明

Qt Creator 内置的最小化示例插件。仅在 IDE 菜单栏添加一个 "Hello World!" 菜单项，点击后无实质操作。作为插件开发教学模板使用，默认禁用（`DisabledByDefault: true`）。仅在开发模式（非 `IDE_PACKAGE_MODE`）下构建。

**裁剪原因**：纯教学示例，对实际功能无贡献。

#### 依赖关系

```
QTC_PLUGIN_DEPENDS: coreplugin
QTC_LIB_DEPENDS:    extensionsystem
QTC_PLUGIN_RECOMMENDS: （无）
反向依赖:           无
```

#### 代码结构

```
src/plugins/helloworld/              （7 个文件，284 行 C++）
├── helloworld_dependencies.pri      # 依赖声明
├── helloworld.pro                   # 构建配置
├── HelloWorld.json.in               # 插件元数据
├── helloworldplugin.h               # 插件主类头文件
├── helloworldplugin.cpp             # 插件主类实现（注册菜单 Action）
├── helloworldwindow.h               # Hello World 窗口头文件
└── helloworldwindow.cpp             # Hello World 窗口实现
```

#### 删减方法

```
修改文件：src/plugins/plugins.pro
操作：删除以下条件编译块中的 helloworld 行
    isEmpty(IDE_PACKAGE_MODE) {
        SUBDIRS += \
            helloworld          ← 删除此行
    }

物理删除：rm -rf src/plugins/helloworld/
```

**注意**：删除 helloworld 后，如果 `isEmpty(IDE_PACKAGE_MODE)` 条件块为空，应将整个条件块删除。

### 2.5 serialterminal — 串口终端工具

**裁剪原因**：HldPlugin 为通用桌面 IDE，不集成硬件串口通信功能。

### 2.7 emacskeys — Emacs 按键绑定

### 2.9 scxmleditor — SCXML 状态机编辑器

#### 功能说明

SCXML（State Chart XML）文件的可视化编辑器。提供图形化界面创建和编辑 SCXML 状态机文档。支持：

- 图形化布局和层级导航
- 颜色主题和缩放
- SCXML 代码与图形双向同步
- 注册 `application/scxml+xml` MIME 类型（`*.scxml`）

**裁剪原因**：状态机编辑是专用工具功能，HldPlugin 为通用插件化 IDE 框架，不需要 SCXML 编辑能力。这是第一批中文件数最大的插件（214 个文件）。

### 2.23 modelinglib — UML 建模库

#### 功能说明

UML 建模的底层基础库，为 `modeleditor` 插件提供完整的图表模型、渲染、序列化、样式等基础设施。包含三个子模块：

1. **qmt (Qt Modeling Toolkit)** — 核心建模工具包（约 290 个文件）
   - `config/` — 配置管理
   - `controller/` — 控制器层
   - `diagram/` — 图表数据模型（类图、包图、组件图）
   - `diagram_controller/` — 图表控制器
   - `diagram_scene/` — 图形场景（Qt Graphics View 框架）
     - `capabilities/` — 能力接口
     - `items/` — 图形项（类项、包项、关系项等）
     - `parts/` — 图形部件
   - `diagram_ui/` — 图表 UI
   - `diagram_widgets_ui/` — 图表控件 UI
   - `document_controller/` — 文档控制器
   - `infrastructure/` — 基础设施（UID、Handle、异常等）
   - `model/` — 元模型（MObject、MClass、MPackage 等）
   - `model_controller/` — 模型控制器
   - `model_ui/` — 模型 UI
   - `model_widgets_ui/` — 模型控件 UI
   - `project/` — 项目管理
   - `project_controller/` — 项目控制器
   - `resources/` — 图标资源（25x25、48x48）
   - `serializer/` — 模型序列化器
   - `stereotype/` — UML Stereotype 管理
   - `style/` — 样式系统
   - `tasks/` — 异步任务

2. **qtserialization** — Qt 序列化框架（约 20 个文件）
   - 模板化的 XML/文件序列化引擎
