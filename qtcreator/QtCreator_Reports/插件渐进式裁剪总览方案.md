## 一、工程基本设定

### 1.1 跨平台支持矩阵

本工程为**跨平台工程**，裁剪时必须兼顾全部目标平台，不得引入平台特定的破坏性变更。

| 操作系统 | 版本 | 硬件架构 |
|----------|------|----------|
| Windows | Windows 7 / Windows 10 / Windows 11 | x86_64, x86_32 |
| 银河麒麟 | Kylin V10 | x86_64, Arm64 |
| 中标麒麟 | NeoKylin V7 | x86_64, Arm64 |
| Ubuntu | 18.04 / 20.04 / 22.04 | x86_64, x86_32, Arm64, Arm32 |

**裁剪原则**：
- 所有裁剪操作必须在 `.pro` / `.pri` 层面完成，不得引入平台条件编译差异
- 保留 `linux-*` 条件块（如 `debugger/ptracepreload.pro`）
- 保留 `win32:` 条件块（如 `utils/process_ctrlc_stub.pro`）
- 保留 `msvc:` 条件块（如 `qthldplugincdbext`）
- 删除插件时，若该插件内部有平台条件编译，整个插件目录一并删除即可

## 四、渐进式裁剪方案

### 4.1 裁剪原则

1. **每批裁剪后必须编译验证**——在 Windows 上使用 qmake + MSVC 构建，确认 0 error
2. **只改构建入口，不删源码目录**——第一阶段从 `plugins.pro` 的 SUBDIRS 移除即可
3. **源码目录删除放在验证通过后**——二次提交时再做物理删除
4. **每批裁剪出一份执行记录**——格式参照 `qt-creator-4.13.3-plugin-trim-record-001-studiowelcome.md`

## 六、每批裁剪的执行流程

每批裁剪均按以下标准流程执行：

```
1. 在 plugins.pro 的 SUBDIRS 中注释掉（或删除）目标插件
2. 如有配套库需移除，同步修改 libs.pro
3. 在 Windows 上执行 qmake + 全量构建，确认 0 error 0 warning（新增）
4. 在目标 Linux 平台（如有条件）交叉验证
5. 编译验证通过后，物理删除对应的源码目录
6. 输出裁剪执行记录文档（格式参照 001-studiowelcome 记录）
7. 提交版本控制
```
