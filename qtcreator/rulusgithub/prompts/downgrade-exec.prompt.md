# 代码降级改造

根据已有的兼容性审计报告，执行实际的代码改造。

## 输入参数
- **审计报告**: ${input:report:参考的审计报告文件名}
- **改造范围**: ${input:scope:执行范围，如 polyfill/QOverload替换/全部}

## 执行步骤

1. 读取指定的审计报告，提取改动清单
2. 按报告中的推荐顺序执行改造：
   - 第一步：创建 polyfill 头文件
   - 第二步：修改版本门槛
   - 第三步：脚本化批量替换
   - 第四步：手工修改少量文件
3. 每步完成后验证编译（如有环境）
4. 汇总实际改动统计

## 约束
- 所有修改添加 `// Qt56Compat:` 注释标记
- 不修改 3rdparty 库核心逻辑
- 保持双平台（MSVC2015 + GCC）兼容
