# 报告格式规范

## 结构要求
- 标题使用一级标题，含报告主题和版本号
- 必须包含"基于...代码审计"的数据来源说明
- 使用表格呈现对比数据和统计结果
- 改动统计必须包含：文件数、出现次数、严重度、替代方案

## 内容要求
- 禁止空洞总结和套话
- 每个结论必须有代码证据支撑（文件路径 + 行号）
- API 兼容性表格必须标注引入版本
- 可行性报告必须给出工作量估算（天数）和推荐实施顺序
