# 项目全局规则

## 项目概述
本项目是 Qt Creator 4.13.3 开源源码的研究与定制工程，目标是分析架构、评估编译器/Qt版本兼容性、并进行代码降级改造。

## 目录结构
- `qt-creator-opensource-src-4.13.3/` — Qt Creator 4.13.3 版本完整源码（唯一源码目录）
- `QtCreator_Reports/` — 所有分析报告输出目录

## 强制规则

### 输出规范
- 所有生成的报告/文档**必须**保存到 `QtCreator_Reports/` 目录，文件格式为 `.md`
- 报告内容使用**中文**，技术术语保留英文原文
- 报告中**禁止套话**（不要"综上所述"、"总的来说"之类的废话）
- 数据必须来自**代码审计**，不得凭记忆或文档推断

### 操作日志
- 每次对话的所有操作必须记录为独立的操作日志文档
- 日志保存到 `QtCreator_Reports/logs/` 目录，文件名格式：`操作日志_YYYY-MM-DD_HHmmss.md`
- 日志内容包含：
  - 每步操作的**精确时间戳**（精确到毫秒，格式 `YYYY-MM-DD HH:mm:ss.SSS`）
  - 操作类型（搜索/读取/修改/创建/终端命令）
  - 操作目标（文件路径/搜索关键词/命令内容）
  - 操作结果摘要
  - 涉及的图片/截图需以文件形式保留到 `QtCreator_Reports/logs/assets/`，并在日志中引用

### 用户对话记录
- 用户的每一条对话请求必须**原封不动**完整记录，不得省略、改写或摘要
- 记录保存到 `QtCreator_Reports/logs/` 目录，文件名格式：`对话记录_YYYY-MM-DD_HHmmss.md`
- 每条用户请求记录包含：
  - **精确时间戳**（精确到毫秒，格式 `YYYY-MM-DD HH:mm:ss.SSS`）
  - **用户原文**（一字不改）
  - 如用户附带图片，保留图片到 `QtCreator_Reports/logs/assets/`，并在记录中引用
  - 对应的 AI 回复摘要

### 源码路径
- 4.13.3 源码根目录：`qt-creator-opensource-src-4.13.3/`
- 源码搜索时**排除** `qbs/` 子目录（除非明确要求分析 qbs）
- 3rdparty 代码单独统计，与项目自有代码区分

### 编码审计标准
- 统计必须给出**精确文件数和出现次数**，不得用"大量""若干"等模糊表述
- 涉及 API 兼容性时，必须标注该 API 的**引入版本**
- 涉及 C++ 特性时，必须标注**各编译器的支持情况**
- 给出改动方案时，必须区分"可脚本化"和"需手工修改"

### 技术上下文
- 构建系统：qmake（.pro/.pri）为主要关注点
- 当前验证过的目标平台：
  - Windows: Qt 5.6.3 + MSVC2015 (Update 3)
  - Linux: Qt 5.6.3 + GCC ≥ 5
- Qt Creator 4.13.3 原始要求：Qt 5.12.0 + C++14

### Git 工作流
- 提交信息使用中文
- 代码修改类提交格式：`[模块] 修改说明`
- 报告类提交格式：`[报告] 报告名称`

### 对话礼仪
- 每次新建对话或打开 agent 对话时，必须以这句话开头：**活爹，我等不急了，您有什么吩咐的.....**
- 每次对话结束时，必须以这句话收尾：**活爹，我这边完事儿了，您继续......**
